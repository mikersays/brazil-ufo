# CLAUDE.md — brazil-ufo

Static archive of 493 Brazilian UFO case reports (1952–2015) scraped from
brazilianufoarchives.com, served as a GitHub Pages site.

- **Live site:** https://mikersays.github.io/brazil-ufo/ (source: `main` → `/docs`)
- **Pipeline:** `tools/extract.py` (pulls posts + photos) → swarm enrichment →
  `tools/merge_enrichment.py` (folds metadata into the catalog).
- **Site root is `docs/`** — `docs/index.html` + `docs/data/`. The tools write to
  `docs/data/` (see the `DATA` constant in each script).
- Source is WordPress.com-hosted: the direct `/wp-json/` 404s; use the public API
  `public-api.wordpress.com/rest/v1.1/sites/brazilianufoarchives.wordpress.com/posts/`
  (already wired into `tools/extract.py`).

## Gotchas (hard-won — don't re-derive)

1. **Post-index gaps.** Real posts are numbered 1–495 but only **493 exist**. Two
   `__trashed-*` posts consumed indices 492 & 493 (filtered out, no folders), while
   real posts run *through* 494 & 495 (the Barra da Tijuca case). So "missing 492/493"
   is expected and correct; do **not** try to backfill them.
   - When fanning out enrichment/processing, iterate over the **actual** post indices
     from `docs/data/index.json` (max index = 495), **not** `range(1, count+1)`. The
     original swarm capped at 493 and silently skipped real posts 494/495 — those were
     hand-filled afterward.

2. **GitHub Pages renders the README instead of the app?** That means Jekyll ran or the
   source isn't `/docs`. Fix: ensure `docs/.nojekyll` exists and Pages source is
   `main`→`/docs`, then force a rebuild:
   `gh api -X POST repos/mikersays/brazil-ufo/pages/builds`. CDN can cache the old
   build for a few minutes; verify with a `?v=N` cache-buster.

3. **Verifying the site in a browser.** A local `python3 -m http.server` started from
   the agent sandbox is **not reachable from the user's Chrome** (separate network
   namespace) — screenshots come back as "Frame is showing error page." Use the
   **Playwright MCP** browser for verification instead; it shares the sandbox network.
   Serve with `python3 -m http.server 8787 --directory docs`.

## Notes

- The structured metadata (year, location, shape, occupants, effects, summaries) is
  machine-generated by the enrichment swarm and **may contain errors** — the verbatim
  source transcript is always in each post's `content.html` / `post.md`.
